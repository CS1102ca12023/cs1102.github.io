Gif

Gif use LZW (Lempel-Ziv-Welch) compression algorithm. The compression contain one stages which is encoding.

The benefits of gif file format:

  • Image with large area of one colour can result in greater compression
  • Image havaing vertical color changes can create less image size

For example: The simple image on the left includes a large amount of one color, leading to a smaller image size. The complex image on the right consists of many colors, resulting in a larger image size.

The disadvantages of gif file format:

  • Limited to a 256-color palette. Information of image will lost if image enclose more than 256 color
  • LZW is difficult to compress complex image, like full-color images

Encoding

LZW compression make use of a code table form 1 to 255 codes.

Encoding process

  1. Dictionary Initialization: The dictionary need to contain all poissable code
  2. Read Input Data: It will read the input data from first character as the current sequence
  3. Find the Longest Match: It look for the dictionary for the longest sequence that matches the current input. If there is a match exist, the current sequence is extended by puting next character, and continues seek for an even longer match
  4. Output Codes: When there is an unmatched sequence, the algorithm outputs the code according to the longest matching sequence
  5. Update the Dictionary: Add current sequence and next character into dictionary, then assigne it to next available code
  6. Repeat: Rerun step 2 to step 5 until processed all input
